Creating Views in Django

Views in Django are Python functions or classes that receive web requests and return web responses. Views access data through models and delegate formatting to the templates.

Function-Based Views

Function-based views are simple Python functions that take a web request and return a web response. Here's an example:

from django.http import HttpResponse

def my_view(request):
    return HttpResponse('Hello, World!')

Class-Based Views

Class-based views provide an object-oriented approach to views. They allow you to organize your views and reuse code by inheriting from Django's generic views. Example:

from django.views.generic import ListView
from .models import MyModel

class MyListView(ListView):
    model = MyModel

URL Configuration

To make a view accessible through the web, you must map it to a URL. In your Django app, modify the urls.py file to include your view:

from django.urls import path
from .views import my_view, MyListView

urlpatterns = [
    path('', my_view, name='home'),
    path('list/', MyListView.as_view(), name='my-list-view'),
]

With these configurations, Django knows how to route incoming web requests to the appropriate view based on the requested URL.
