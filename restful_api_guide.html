Building RESTful APIs with Django

Expanded View Examples

GET Request Endpoint

This view handles GET requests for fetching all instances of MyModel.

from rest_framework.generics import ListAPIView
from .models import MyModel
from .serializers import MyModelSerializer

class MyModelListView(ListAPIView):
    queryset = MyModel.objects.all()
    serializer_class = MyModelSerializer

GET and POST Request Endpoint

This view can handle both GET requests to list instances and POST requests to create a new instance of MyModel.

from rest_framework.generics import ListCreateAPIView
from .models import MyModel
from .serializers import MyModelSerializer

class MyModelListCreateView(ListCreateAPIView):
    queryset = MyModel.objects.all()
    serializer_class = MyModelSerializer

GET, POST, PUT, and DELETE Endpoint

This view supports GET for retrieving, POST for creating, PUT for updating, and DELETE for deleting instances of MyModel.

from rest_framework.generics import RetrieveUpdateDestroyAPIView
from .models import MyModel
from .serializers import MyModelSerializer

class MyModelDetail(RetrieveUpdateDestroyAPIView):
    queryset = MyModel.objects.all()
    serializer_class = MyModelSerializer

URL Configuration for Expanded Examples

Finally, ensure your urls.py maps URLs to these new views:

from django.urls import path
from .views import MyModelListView, MyModelListCreateView, MyModelDetail

urlpatterns = [
    path('api/my-model/', MyModelListView.as_view(), name='my-model-list'),
    path('api/my-model/create/', MyModelListCreateView.as_view(), name='my-model-create'),
    path('api/my-model/detail//', MyModelDetail.as_view(), name='my-model-detail'),
]

These examples showcase different ways to set up your API endpoints to handle various HTTP methods, allowing for a flexible and powerful API design.
